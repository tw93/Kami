01 · ABOUT
Kaku란
Kaku(書く)는 생각을 형태로 빚는 행위를 뜻하는 말입니다. WezTerm을 기반으로 깊이 최적화하여 실용적인 기본값, 완전한 Lua 커스터마이징, 가볍고 쾌적한 조작감을 동시에 갖춘 터미널입니다.

주력 플랫폼

Kaku는 3부작 중 하나입니다. Kaku(書く)가 코드를 쓰고, Waza(技)가 습관을 단련하며, Kami(紙)가 문서를 완성합니다. 오랫동안 CLI 중심으로 일해 왔고, Alacritty를 수년간 사용했습니다. AI 보조 코딩으로 무게 중심이 옮겨가면서 더 강력한 탭 관리와 내장 AI 연동이 필요해졌습니다. Kitty, Ghostty, Warp, iTerm2도 써봤지만, 성능 우선 · 기본값으로 쓸 수 있음 · Lua로 전면 확장 가능이라는 세 가지 균형에 가장 가까운 것이 Kaku였습니다.

WezTerm은 강력한 엔진입니다. Kaku는 그 위에서 두 가지에 집중했습니다. 불필요한 무게를 줄이고, AI 워크플로를 직접 통합하는 것입니다. 설정은 WezTerm Lua API와 호환되어 기존 설정을 그대로 재사용할 수 있습니다. Starship, Delta, Lazygit, Yazi도 처음부터 연동되어 있습니다. macOS에서는 Traffic Light 탭바 통합, 시스템 외관 자동 전환, 네이티브 폰트 렌더링, Apple 공증까지 포함하여 설치 직후부터 실용 상태가 됩니다.

02 · FEATURES
주요 기능
Kaku는 복잡한 초기 설정 없이 바로 사용할 수 있습니다. 동시에 WezTerm Lua API 호환을 유지해 깊은 커스터마이징도 지원합니다.

제로 설정 시작

기본 폰트는 JetBrains Mono. macOS 렌더링 특성에 맞게 조정되어 있습니다. 디스플레이 해상도를 자동 감지해 저해상도는 15px, 고해상도는 17px, 행높이 1.28을 자동 적용합니다.

테마 자동 전환

macOS 외관 설정에 연동해 Kaku Dark / Kaku Light를 자동으로 전환합니다. 선택 색상, 굵기, 색상 오버라이드 테이블도 함께 업데이트되며, color_scheme으로 고정 운용도 가능합니다.

엄선된 셸 도구

zsh 플러그인을 자동으로 불러옵니다. z 디렉토리 이동, zsh-completions, 구문 강조, 자동 완성 후보, Smart Tab을 지원하며 fish 연동도 가능합니다.

빠르고 가볍게

심볼 축소와 기능 정리로 바이너리를 40% 압축했습니다(67MB → 40MB). 리소스는 100MB에서 80MB로. JIT 초기화로 셸 초기화 시간도 절반으로 줄었습니다(200ms → 100ms).

WezTerm 호환 설정

WezTerm의 Lua 설정을 그대로 사용할 수 있어 마이그레이션이 필요 없습니다. kaku.lua는 기본값을 먼저 읽은 뒤 오버라이드를 적용합니다.

세련된 기본값

선택 복사, 클릭 가능한 경로, 히스토리 표시, 페인 입력 브로드캐스트, 백그라운드 완료 알림, Traffic Light 탭바 통합을 기본으로 제공합니다.

Lazygit 연동

PATH 또는 Homebrew 경로에서 lazygit 바이너리를 자동 감지합니다. 미커밋 변경사항이 있을 때 한 번만 알림을 보냅니다.

Cmd + Shift + G

Yazi 연동

셸 래퍼 y로 실행하고, 종료 시 작업 디렉토리를 동기화합니다. 배색은 yazi/theme.toml에 자동 반영됩니다.

Cmd + Shift + Y

원격 파일(SSHFS)

활성 SSH 페인에서 연결 대상을 자동 감지하고, sshfs로 원격 파일 시스템을 마운트해 Yazi로 탐색할 수 있습니다.

Cmd + Shift + R
03 · AI ASSISTANT
Kaku AI 어시스턴트
Kaku에는 AI 어시스턴트가 내장되어 있습니다. 두 가지 모드로 터미널 작업의 주요 AI 사용 사례를 커버합니다. kaku ai로 설정 패널을 열어 외부 코딩 도구와 Kaku Assistant를 구성할 수 있습니다.

오류 자동 복구

명령 실패 시, 실패 명령 + 종료 코드 + 작업 디렉토리 + git 브랜치를 LLM에 전송하고, 수정 제안을 터미널 내에 표시합니다. Cmd + Shift + E로 바로 적용할 수 있습니다.

미발동 조건: Ctrl+C 중단, help 플래그, 단순 패키지 매니저 실행, git pull 충돌, 비셸 전경 프로세스. 위험 명령(rm -rf, git reset --hard)은 붙여넣기 가능하지만 자동 실행하지 않습니다.

자연어로 명령 생성

프롬프트에서 # <설명>을 입력하고 Enter를 누르면, Kaku가 셸에 전달하기 전에 해당 줄을 가로채 현재 디렉토리와 git 브랜치와 함께 LLM에 전송합니다. 생성된 명령은 프롬프트로 돌아오며, 확인 후 실행할 수 있습니다.

# 최근 7일간 수정된 파일 목록 출력
# 포트 3000의 프로세스 찾아서 종료
# node_modules 제외하고 src 폴더 압축

04 · PERFORMANCE
성능 비교
심볼 축소, 기능 정리, 리소스 최적화를 통해 Kaku는 기능을 유지하면서도 크기와 시작 시간을 크게 개선했습니다. macOS 전용 설계로 플랫폼 통합도 깊어졌습니다.
지표 상위 WezTerm Kaku 최적화 방법
실행 파일 크기 ~67 MB ~40 MB 심볼 축소 + 기능 정리
리소스 용량 ~100 MB ~80 MB 리소스 최적화 + 지연 로드
시작 지연 표준 즉시 JIT 초기화
셸 초기화 시간 ~200ms ~100ms 환경 설정 플로우 최적화

바이너리 최적화

컴파일 시 미사용 feature flag를 제거하고, 심볼 스트리핑과 디버그 정보 축소를 적용했습니다. Rust release profile은 LTO + codegen-units=1로 최적화를 최대화합니다.

지연 로드 전략

셸 도구군(Starship, Delta, Yazi 테마 동기화)은 just-in-time 초기화를 채택했습니다. 첫 번째 Kaku 세션에서만 시작되어 시스템 전체의 셸 시작 시간에 영향을 주지 않습니다.

macOS 통합

Traffic Light 탭바 통합(INTEGRATED_BUTTONS|RESIZE), 시스템 외관 연동, Apple 공증(경고 없음), macOS 네이티브 폰트 렌더링 스택을 제공합니다.

05 · INSTALLATION
설치와 사용법
Kaku는 간단한 설치 절차와 직관적인 단축키 체계를 제공합니다. 첫 실행 시 셸 환경도 자동으로 설정됩니다.
1

설치

GitHub Releases에서 최신 DMG를 받아 응용 프로그램 폴더로 이동하거나, Homebrew로 설치합니다.

brew install tw93/tap/kaku
2

첫 실행

Kaku는 Apple 공증을 완료했습니다. 보안 경고 없이 실행되며, 첫 실행 시 셸 환경을 자동으로 설정합니다.

3
