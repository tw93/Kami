01 · ABOUT
关于 Kaku
Kaku（書く，かく）是日语"书写"的意思，代表将思想转化为形式的行为。这是一个深度定制的 WezTerm 分支，专注于实用的默认配置，同时保留完整的 Lua 自定义能力和快速轻量的体验。

Kaku 是三部曲的一部分：Kaku（書く）写代码，Waza（技）练习惯，Kami（紙）出文档。可以把它们想象成一个家庭：Kaku 是父亲，Waza 是大姐，Kami 是小妹。三者独立运作，又彼此呼应。

我长期依赖命令行工作，构建的工具如 Mole 和 Pake 都反映了这一点。我使用 Alacritty 多年，学会了重视速度和简洁。随着工作流转向 AI 辅助编码，我需要更强的标签页和窗格管理，以及内置的 AI 交互能力。

WezTerm 是强大的引擎，Kaku 在其基础上做了两件事：大幅裁减不必要的体积，以及把 AI 工作流直接嵌入终端。不再需要在终端、浏览器、IDE 之间反复切换。

我尝试过 Kitty、Ghostty、Warp 和 iTerm2，每个都有各自的优势，但都不是我想要的那个平衡点：性能优先、默认即好用、保留完整的 Lua 可定制性。

Kaku 的配置体系完全兼容 WezTerm Lua API，存量配置不需要迁移。同时预置了 Starship 提示符、Delta diff 高亮、Lazygit、Yazi 等工具的集成，开箱后的终端已经是一个完整的工作环境。

macOS 专属设计意味着 Kaku 可以深度利用平台特性：Traffic Light 嵌入标签栏、系统 Deep Dark 主题跟随、原生字体渲染、Apple 公证开箱即用。

02 · FEATURES
核心特性
Kaku 提供开箱即用的体验，无需复杂配置即可开始工作。同时保留完整的 WezTerm Lua API 兼容性，满足深度定制需求。

零配置启动

默认使用 JetBrains Mono 字体，macOS 字体渲染优化。自动检测显示器分辨率：低分辨率 15px，高分辨率 17px，行高 1.28，无需任何手动设置。

主题自动切换

跟随 macOS 系统外观自动切换 Kaku Dark / Kaku Light，同步更新选区颜色、字重、颜色覆盖表。也可通过 color_scheme 强制锁定任一模式。

精选 Shell 套件

内置 zsh 插件自动加载：z 目录跳转、zsh-completions、语法高亮、自动建议、Smart Tab 智能补全。也支持 fish shell 集成。

快速轻量

激进符号剥离 + 功能精简后二进制减小 40%（67MB→40MB），资源文件从 100MB 降至 80MB。即时初始化，Shell 引导时间减半（200ms→100ms）。

WezTerm 兼容配置

直接使用 WezTerm 的 Lua 配置，完整 API 兼容，无需迁移。kaku.lua 先加载内置 defaults，再应用用户 overrides，结构清晰不冲突。

精致的默认设置

选中即复制、可点击文件路径、全屏应用历史记录查看、窗格输入广播、后台标签页完成视觉提示、Traffic Light 嵌入标签栏。

Lazygit 集成

自动从 PATH 或 Homebrew 路径检测 lazygit 二进制。有未提交改动时显示一次性提示。

Cmd + Shift + G

Yazi 文件管理器

Shell wrapper y 启动 Yazi 并在退出时同步工作目录。自动同步 Kaku 配色到 yazi/theme.toml。

Cmd + Shift + Y

Remote Files（SSHFS）

从活跃 SSH 窗格自动检测目标主机，通过 sshfs 挂载远程文件系统并在 Yazi 中打开浏览。

Cmd + Shift + R
03 · AI ASSISTANT
Kaku AI 助手
Kaku 内置 AI 助手，两种模式覆盖终端最高频的 AI 需求。运行 kaku ai 打开设置面板，配置外部编码工具和 Kaku Assistant。

错误自动恢复

命令失败时自动发送：失败命令 + 退出码 + 工作目录 + git 分支 → LLM，结果内联显示在终端。按 Cmd + Shift + E 直接应用修复。

不触发的场景：Ctrl+C 退出、帮助 flag、裸包管理器调用、git pull 冲突、非 shell 前台进程。危险命令（rm -rf、git reset --hard）粘贴但从不自动执行。

自然语言转命令

在提示符输入 # <描述> 并回车，Kaku 在 shell 看到之前拦截这一行，连同当前目录和 git 分支发给 LLM，将生成的命令注入回提示符供审查后运行。

# list all files modified in the last 7 days
# find and kill the process on port 3000
# compress the src folder excluding node_modules

04 · PERFORMANCE
性能对比
通过激进的符号剥离、功能精简和资源优化，Kaku 在保持完整功能的同时显著减少体积和启动时间。macOS 专属设计带来平台级深度集成。
指标 上游 WezTerm Kaku 优化方法
可执行文件大小 ~67 MB ~40 MB 激进符号剥离 + 功能精简
资源文件体积 ~100 MB ~80 MB 资源优化 + 延迟加载
启动延迟 标准 即时 即时初始化（JIT）
Shell 引导时间 ~200ms ~100ms 优化的环境配置流程

二进制精简

编译时裁掉上游未使用的 feature flags，激进 symbol stripping，移除调试符号。Rust release profile 开启 LTO + codegen-units=1 最大化优化。

延迟加载策略

Shell 套件（Starship、Delta、Yazi 主题同步）采用 just-in-time 初始化，仅在首次进入 Kaku session 时触发，不影响系统级 shell 启动时间。

macOS 平台集成

Traffic Light 嵌入标签栏（INTEGRATED_BUTTONS|RESIZE）、系统外观跟随、Apple 公证（无需安全警告）、macOS 原生字体渲染栈。

05 · INSTALLATION
安装与使用
Kaku 提供简单的安装方式和直观的快捷键系统。首次启动时会自动设置 shell 环境。
1

下载安装

从 GitHub Releases 下载最新的 DMG 文件并拖入应用程序文件夹，或使用 Homebrew 安装：

brew install tw93/tap/kaku
2

首次启动

打开 Kaku，应用已通过 Apple 公证，无需安全警告。首次启动时 Kaku 会自动设置 shell 环境。

3

配置定制

按 Cmd + , 打开设置面板，或运行 kaku config 编辑 Lua 配置文件。
